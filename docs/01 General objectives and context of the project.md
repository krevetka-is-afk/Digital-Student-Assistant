# 1. Общие цели и контекст проекта

Актуализировано: 2026-04-28.

## Цель

Digital Student Assistant предназначен для поддержки выбора и сопровождения студенческих проектов: от импорта проектных предложений и публикации каталога до подачи заявок, модерации, рекомендаций и последующей синхронизации данных с downstream-сервисами.

Текущий источник правды фактическое состоянию проекта: код, `README.md`, `docs/api_scheme.md`, `docs/architecture/contracts/*.json`, тесты и runbook-документы.

## Проблема

Исходный процесс выбора проектов опирается на большие таблицы и внешние формы. Это создает несколько повторяющихся проблем:

- студентам трудно быстро найти релевантные темы среди сотен строк;
- данные переносятся вручную между таблицами, формами и административными процессами;
- статусы заявок и модерации не видны всем ролям в едином контуре;
- рекомендации и поиск трудно поддерживать без единой нормализованной модели проектов;
- внешние ML/graph-компоненты не должны напрямую зависеть от внутренней БД основного backend.

## Пользователи

- `student` - просматривает каталог, управляет интересами и избранным, получает рекомендации, подает заявки и отслеживает статусы.
- `customer` - создает и редактирует проекты, отправляет их на модерацию, смотрит и рассматривает заявки на свои проекты.
- `cpprp` - модерирует проекты, управляет дедлайнами и шаблонами, видит административные сводки и экспортирует данные.
- `staff` - технический/административный override для Django admin и API-поверхностей, где это явно разрешено.

## Фактический runtime

- `src/web` - Django + DRF backend и Django templates frontend.
- `src/ml` - reference FastAPI ML service для поиска и рекомендаций.
- `src/graph` - FastAPI graph projector с Neo4j.
- `apps.faculty` - зеркало данных преподавателей, публикаций, курсов и matching научных руководителей проектов.
- `infra` - Docker Compose, Nginx, deployment и backup/restore scripts.

## Текущий scope

В текущем состоянии реализованы:

- login с подтверждением email;
- role-based кабинеты и API для `student`, `customer`, `cpprp`;
- каталог проектов, карточки, заявки, избранное, интересы;
- импорт EPP из XLSX через API и CLI;
- workflow инициативных тем;
- recommendation/search facade с fallback-режимом при недоступности semantic ML;
- outbox delivery API, checkpoint/ack/replay и bootstrap snapshot;
- reference ML и graph consumers;
- faculty mirror и project-supervisor matching;
- production/staging Docker Compose, deployment runbook, backup/restore и observability docs.
