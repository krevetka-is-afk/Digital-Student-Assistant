# Документация проекта

Актуализировано: 2026-04-29.

## Профиль документации

Документация проекта Digital Student Assistant ведется как смешанный комплект:

- материалы курсового проекта и пояснительной записки, ориентированные на ГОСТ 7.32-2017;
- программная документация, близкая к ЕСПД ГОСТ 19, прежде всего техническое задание;
- инженерные справки для разработки, проверки, эксплуатации и интеграции сервисов.

Если методические указания кафедры или руководителя задают другой порядок разделов, они имеют приоритет над общей структурой этого каталога. ГОСТ используется как опорная схема оформления и содержания.

## Основные документы

| `schema_prjects.md` | схемы входных данных для проектных предложений |

## Как использовать комплект

Для пояснительной записки к курсовому проекту целесообразно использовать документы `01`-`05`, `structure_of_project.md`, `api_scheme.md`, ADR и техническое задание как исходные материалы. Их не следует переносить в отчет дословно: в пояснительной записке требуется связный академический текст, а не набор внутренних заметок.

Для проверки реализации используются:

- контрактные JSON-документы из `architecture/contracts/`;
- тесты из `tests/contract/`, `tests/integration/`, `tests/e2e/`;
- команды проверки, приведенные в `README.md`.

## Недостающие сведения для полного отчета

Перед подготовкой финальной версии курсовой работы нужно отдельно заполнить:

- данные титульного листа по методическим указаниям;
- объект, предмет, цель и задачи работы в формулировках руководителя;
- обзор аналогов и источников;
- описание выбранных технологий с обоснованием;
- результаты испытаний с датой, окружением, наборами данных и фактическими результатами;
- список использованных источников в едином библиографическом стиле.
