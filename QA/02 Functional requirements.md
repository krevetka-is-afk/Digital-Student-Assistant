### 2. Функциональные требования (что система должна делать)

5. **Какие функции точно должны быть в MVP?** (личный кабинет, создание/управление проектами, поиск/фильтрация/рекомендации, подача заявок, управление статусами, граф заявок/проектов, контрольные точки, дедлайны, интеграции с внешними системами, импорт/экспорт данных, …).
6. **Какие функции могут быть «опциональными» / «плюшками» для будущих версий?** (например, мобильный клиент, опросы, расширенная аналитика, интеграция с LMS, уведомления, рекомендации на основе успеваемости, дедлайн‑напоминания).
7. **Какие роли пользователей и права доступа нужны?** Например, студент, преподаватель, администратор, внешний заказчик — и что каждый из них может делать.
8. **Каковы бизнес‑процессы и workflow для ключевых сценариев?** Например: создание проекта → подача заявок → утверждение → участие → завершение → сдача отчёта/вывод списка, дедлайны, контрольные точки и т.п.
