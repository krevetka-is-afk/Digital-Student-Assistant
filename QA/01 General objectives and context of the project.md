# 1. Общие цели и контекст проекта

1. **Какова главная цель и ценность проекта?** Мы хотим дать студентам сервис, который улучшит процесс выбора проектов для КР / ВКР. Благодаря:
   - Системе рекомендаций
   - Семантическому поиску
   - Удобной фильтрации по технологиям / направлениям

2. **Какие ключевые проблемы / боли решает система?**
   1. На сегоднящний день, процесс выбора выглядит так:
      1. Осенью от ЦППРП студентам ФКН направляется таблица с описанием проектов.
      2. Студенты должны с ней ознакомиться (порядка 700-1000 строк/проектов), при этом необходимо внимательно следить к какому курсу и ОП соответствует проект, а также вычитывать описание и процие данные проектов. ==Без "большого экрана" и каких-то знаний excel это сделать проблематично, что может привести к выбору несогласованного для ОП проекта, например==.
      3. Когда понравивишиеся проекты найдены необходимо установить контакт с научными руководителями для утверждения на проект и прохождения отбора (если требуется).
      4. В случае успешной договоренности с научным руководителем, подписывается заявление.
      5. Заявление передается в ЦППРП через форму, в котороый студент переписывает данные из таблицы проектов, сам подбирает подходящие "теги" для опредления в соответствующую комиссию для защиты и прикрепляет заявление. – ==ручной перенос информации / субъективное проставление тегов, потенциальные места ошибок.==
   2. На сегоднящний день, процесс подачи инициативного проекта выглядит так:
      1. Поиск научного руковдителя
      2. Подготока описания проекта
      3. Заполнение формы
      4. Ожидание ответа
   3. На сегоднящний день, процесс подачи проекта от заказчика выглядит так:
      1. Не знаем, как он выглядит

3. **Кто основные пользователи?**:
   1. ЦППРП
   2. Студенты
   3. Внешние / внутренние заказчики

4. **Какие сценарии использования критичны?**
   1. Студент:
      1. Открывает портал
      2. Входит под edu.hse.ru
      3. Может остматривать все проекты с филтрацией и без
      4. Может воспользоваться рекомендательной системой
      5. Может воспользоваться семантическим поиском
      6. Может отсматривать детальную информацию о проекте (включая количество поданных заявок и утвержденных участников)
      7. Может откликнуться на ≥1 проекта
      8. Может получить контактуную информацию заказчика
   2. Внешние / внутренние заказчики
      1. Открывает портал
      2. Входит под edu.hse.ru / или ?
      3. Может создать проект заполнив описание по "гибкому" шаблону
      4. Может просматривать заявки от студентов
      5. Может просмотривать информацию профиля / резюме студента, подавшего заявку
      6. Может примнимать / отклонять заявки студентов
   3. администратор ЦППРП
      1. Открывает портал
      2. Входит под edu.hse.ru
      3. Устанавливает дедлайны для ОП
      4. Устанавливает шаблоны документов для ОП
      5. ?Модерирует заявки от заказчиков?
      6. ?Распределяет проекты по ОП?
